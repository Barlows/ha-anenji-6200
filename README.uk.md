> **Картка-компаньйон для дашборду:** Поєднайте цю інтеграцію з [EyeBond Local Card](https://github.com/groove-max/ha-eybond-local-card), щоб одразу отримати готовий Lovelace-інтерфейс з анімованою схемою потоків енергії та графіками історії.

**EyeBond Local** — інтеграція для Home Assistant, яка напряму працює з гібридними інверторами, підключеними через Wi-Fi-колектори SmartESS / EyeBond.

Ви отримуєте моніторинг у реальному часі, лічильники енергії та керування підтримуваними інверторами через локальну мережу.

Для підтримуваних колекторів можна або зберегти роботу звичайного застосунку SmartESS паралельно з Home Assistant у режимі `SmartESS + HA`, або перевести колектор у `HA only`, якщо ви хочете, щоб цей колектор працював лише з Home Assistant.

Якщо штатний моніторинг вашого інвертора вже працює через SmartESS, це зазвичай найкраща ознака сумісності.

> **Примітка:** Інтеграція активно розвивається. Для вашого інвертора може знадобитися архів підтримки, щоб ми могли підтвердити сумісність і зрозуміти, що робити далі. Дивіться [Підтримуване обладнання](#підтримуване-обладнання) та [Як отримати допомогу](#як-отримати-допомогу).

## Що вміє

- **Повсякденна робота лишається локальною** — Home Assistant читає й записує інвертор через вашу локальну мережу.
- **Керований майстер налаштування** зі швидким і глибоким скануванням, ручним резервним сценарієм і, для підтримуваних колекторів, налаштуванням Wi-Fi через Bluetooth.
- **Два режими колектора** — `SmartESS + HA` зберігає роботу штатного застосунку; `HA only` переводить цей колектор лише в Home Assistant.
- **Безпечно за замовчуванням** — режими `Read-only`, `Auto` і `Full Control` дають змогу вибрати, наскільки широко Home Assistant може змінювати налаштування.
- **Окремі пристрої колектора та інвертора** — інструменти колектора в одному місці, а щоденні сенсори інвертора в іншому.
- **Необов'язкова допомога від хмари SmartESS** — повторно використовувані хмарні дані для діагностики й підтримки, а не для обходу локальних запобіжників безпеки.
- **Налаштовуваний інтервал опитування** — від `2` до `3600` секунд.
- **Готово до панелі енергії Home Assistant** — обчислені підсумки для PV, навантаження, батареї та, на підтримуваних моделях, імпорту/експорту з мережі.

## Підтримуване обладнання

EyeBond Local працює з інверторами, для яких штатний моніторинг доступний через **SmartESS**. Це може бути окремий **Wi-Fi колектор SmartESS / EyeBond** або вбудований Wi-Fi-модуль, що використовує той самий локальний протокол.

| Модель / клас обладнання | Стан | Що це означає |
|---|---|---|
| **Sandisolar SD-HYM-4862HWP** | Підтримується | Повний моніторинг і перевірене керування на підтвердженому шляху SMG. |
| **Anenji ANJ-11KW-48V-WIFI-P** | Підтримується | Повний моніторинг, телеметрія PV1/PV2, підтримка годинника інвертора та перевірене керування. |
| **Anenji 4200 (Protocol 1)** | Часткова підтримка | Моніторинг працює, але можливості запису все ще потребують ширшої перевірки на реальному обладнанні. Використовуйте `Full Control` лише якщо розумієте ризик. |
| **PowMr 4.2kW** (сирий ідентифікатор `VMII-NXPW5KW`) | Підтримується | Повний моніторинг і перевірене керування на підтвердженому шляху родини PI30. |
| **Невідомий, але явно інвертор родини SMG** | Резервний режим лише для читання | Моніторинг лишається доступним, але запис навмисно вимкнений, доки точну модель не буде підтверджено. |
| **Обладнання родини PI18** | Експериментально | Перевірено лише на збережених даних відтворення. Корисно для досліджень, але ще не готово як публічна робоча підтримка. |

Для глибших нотаток щодо протоколів і деталей по родині SMG дивіться [матрицю підтримки SMG](docs/SMG_SUPPORT_MATRIX.md).

Не бачите свого інвертора? Він усе одно може запрацювати — створіть звернення з [архівом підтримки](#як-отримати-допомогу), і ми зможемо оцінити сумісність та, якщо протокол збігається, розширити підтримку.

## Встановлення

### Через HACS (рекомендовано)

1. Відкрийте **HACS → Інтеграції**.
2. Натисніть меню (три крапки) → **Користувацькі репозиторії**.
3. Додайте `https://github.com/groove-max/ha-eybond-local` у категорії **Інтеграція**.
4. Знайдіть **EyeBond Local** у списку та натисніть **Завантажити**.
5. Перезапустіть Home Assistant.
6. Перейдіть у **Налаштування → Пристрої та служби → Додати інтеграцію** і знайдіть **EyeBond Local**.

### Ручне встановлення

1. Завантажте останній реліз.
2. Скопіюйте `custom_components/eybond_local/` у каталог `config/custom_components/` вашого Home Assistant.
3. Перезапустіть Home Assistant.
4. Додайте інтеграцію через **Налаштування → Пристрої та служби**.

## Покрокове налаштування

Майстер налаштування спочатку допомагає підключити колектор, а потім переходить до інвертора.

**1. Ласкаво просимо** — виберіть тип підключення і запустіть процес.

<p align="center"><img src="docs/images/setup-01-welcome.png" alt="Екран привітання" width="320"></p>

**2. Підключіть колектор до тієї самої мережі** — якщо він уже в тій самій Wi-Fi- або LAN-мережі, що й Home Assistant, переходьте далі. Якщо ні, майстер пояснює практичні варіанти: скористатися застосунком SmartESS, зробити це вручну або, на підтримуваних колекторах, передати Wi-Fi-налаштування через Bluetooth.

<p align="center"><img src="docs/images/setup-02-collector-network.png" alt="Вибір способу підключення колектора до мережі" width="480"></p>

Якщо ваш колектор підтримує налаштування через Bluetooth, EyeBond Local може напряму записати назву Wi-Fi-мережі та пароль, а потім повернутися до звичайного мережевого сканування.

<p align="center"><img src="docs/images/setup-03-bluetooth-wifi.png" alt="Налаштування Wi-Fi через Bluetooth" width="480"></p>

**3. Виберіть, як шукати пристрій** — виберіть мережевий інтерфейс Home Assistant, а потім почніть зі швидкого сканування, запустіть глибоке сканування або відразу перейдіть до ручного налаштування.

<p align="center"><img src="docs/images/setup-04-scan-interface.png" alt="Вибір інтерфейсу сканування" width="480"></p>

**4. Проскануйте мережу** — звичайне швидке сканування зазвичай завершується за 5–15 секунд. Якщо воно нічого не знаходить, той самий процес дозволяє повторити спробу, переключитися на глибоке сканування або відкрити ручне налаштування.

<p align="center"><img src="docs/images/setup-05-scanning.png" alt="Сканування мережі" width="480"></p>

**5. Перегляньте знайдені пристрої** — ви побачите знайдені колектори та інвертори з чіткими статусами:

- **Готово** — інтеграція впевнена, що це правильний пристрій, і його безпечно додавати.
- **Перевірити** — пристрій знайдено, але варто ще раз перевірити, що це саме він.
- **Лише колектор** — колектор відповів, але інвертор ще не вдалося точно визначити.

<p align="center"><img src="docs/images/setup-06-detected-devices.png" alt="Виявлені пристрої" width="480"></p>

**6. Підтвердіть і виберіть режим колектора** — перегляньте знайдений колектор та інвертор, виберіть `SmartESS + HA` або `HA only`, а потім завершіть налаштування.

<p align="center"><img src="docs/images/setup-07-confirm.png" alt="Підтвердження виявлення та вибір режиму колектора" width="480"></p>

Якщо швидке й глибоке сканування непрактичні, **Ручне налаштування** все ще доступне і зберігає ті самі розширені мережеві поля, що й раніше. У більшості домашніх сценаріїв ці додаткові поля можна не чіпати й використовувати стандартний варіант для тієї самої локальної мережі.

<p align="center"><img src="docs/images/setup-manual.png" alt="Ручне налаштування" width="360"></p>

> **Порада:** Тримайте Home Assistant і колектор в **одній підмережі**, якщо хочете користуватися автопошуком, бо широкомовне виявлення зазвичай не проходить через роутери.

### SmartESS + HA чи HA only?

EyeBond Local показує два повсякденні режими колектора:

- **`SmartESS + HA`** зберігає видимість колектора у застосунку SmartESS, поки Home Assistant також спілкується з ним локально. Це рекомендований варіант за замовчуванням для більшості користувачів.
- **`HA only`** змушує цей колектор перепідключитися лише до Home Assistant. Обирайте його, якщо хочете перестати покладатися на хмару SmartESS для цього колектора.

Ви можете вибрати режим під час налаштування й змінити його пізніше через **Параметри роботи** або на сторінці пристрою колектора.

У режимі `SmartESS + HA` Home Assistant працює з колектором окремо від застосунку SmartESS. Через це після збоїв Wi-Fi вони не завжди відновлюються одночасно. Іноді колектор ненадовго зникає з локальної мережі та з Home Assistant, хоча застосунок SmartESS ще працює. Коли Wi-Fi знову стає стабільним, колектор зазвичай повертається сам за кілька хвилин.

### Тимчасовий пристрій / EyeBond Setup Pending

Якщо майстер може зберегти робоче підключення до колектора, але ще не може повністю підтвердити інвертор, він усе одно може створити **тимчасовий запис лише для читання**. У Home Assistant він з'являється як **EyeBond Setup Pending**.

Це проміжний стан, а не обов'язково помилка.

Що це означає:

- EyeBond Local уже зберіг підключення до колектора.
- Колектору ще може знадобитися перепідключитися, або інтеграція ще завершує визначення інвертора.
- Діагностика й дії підтримки доступні одразу, навіть якщо звичайні сенсори ще недоступні.
- Налаштування може завершитися пізніше саме, коли колектор перепідключиться або коли ви повторите сканування чи ручну перевірку.

Що робити далі:

1. Зачекайте трохи, а потім оновіть сторінку пристрою. Якщо колектор щойно переналаштовували, його перезавантаження може допомогти.
2. У звичайній локальній мережі тримайте Home Assistant і колектор в одній підмережі, а потім повторіть швидке сканування, глибоке сканування або ручну перевірку.
3. Для ручного чи віддаленого підключення ще раз перевірте IP-адресу колектора, поля **Advertised callback IP** і **Advertised callback TCP port**, а також що TCP `8899` і UDP `58899` не заблоковані.
4. Якщо пристрій і далі лишається у стані очікування, відкрийте **Налаштувати → Діагностика та експериментальні метадані** і створіть **архів підтримки**, перш ніж змінювати розширені налаштування.

### Ручне налаштування для віддаленого підключення / NAT

Більшості користувачів ці розширені поля не потрібні. Вони призначені лише для віддалених колекторів, VPN-з'єднань або сценаріїв із переадресацією портів / NAT.

Якщо Home Assistant і колектор знаходяться в одній локальній мережі, використовуйте стандартний локальний сценарій і залишайте поля зворотного підключення порожніми.

Основні поля:

- **Local listener IP** — адреса Home Assistant, до якої колектор має звертатися локально.
- **Collector IP** — адреса колектора, якщо ви вже її знаєте й хочете перевірити пристрій напряму.
- **Advertised callback IP / TCP port** — лише для VPN або публічних сценаріїв із переадресацією, де колектор має підключатися назад до іншої досяжної адреси.

Потрібен повний покроковий приклад для VPN і переадресації портів? Дивіться [посібник із віддаленого підключення / NAT](docs/REMOTE_SETUP.md).

## Що ви отримаєте

Після налаштування EyeBond Local зазвичай створює **два пристрої Home Assistant** для однієї інсталяції:

- **Пристрій колектора** — IP колектора, якість сигналу, підключена Wi-Fi-мережа, режим роботи колектора, перезапуск, зміна Wi-Fi, проксі-перехоплення трафіку і дії підтримки.
- **Пристрій інвертора** — сенсори інвертора, PV, батареї та мережі в реальному часі, а також підтримувані налаштування й кнопки для цього обладнання.

<p align="center"><img src="docs/images/device-overview.png" alt="Пристрої колектора та інвертора в Home Assistant" width="720"></p>

Для інвертора ви отримуєте:

- **Сенсори** — потужність, напругу, струм, температуру, частоту, режим роботи та інші дані в реальному часі, які віддає обладнання.
- **Лічильники енергії** — обчислені значення `kWh` для виробітку PV, споживання навантаження та заряду/розряду батареї, а також імпорту/експорту з мережі на моделях, що віддають потужність мережі.
- **Бінарні сенсори** — стан заряджання, аварії, помилки та інші прапори стану там, де це підтримується.
- **Підтримуване керування** — сутності `number`, `select`, `switch` і `button` для налаштувань на кшталт лімітів заряду, вихідного режиму, звукового сигналу та окремих дій для конкретної моделі.
- **Керування опитуванням** — можна змінювати, як часто оновлюються сенсори, у межах від `2` до `3600` секунд.

<p align="center"><img src="docs/images/inverter-sensors.png" alt="Сенсори інвертора після налаштування" width="320"></p>

## Режими та налаштування

Після налаштування для більшості користувачів найважливіші два параметри:

- **Режим роботи колектора** вирішує, чи продовжує колектор спілкуватися і з хмарою SmartESS, і з Home Assistant, чи працює лише з Home Assistant.
- **Режим керування** вирішує, скільки прав на запис отримає Home Assistant на стороні інвертора.

`Read-only` лише показує дані. `Auto` сам вмикає перевірені елементи керування, коли інтеграція достатньо впевнена у визначенні пристрою. `Full Control` відкриває всі команди запису для досвідчених користувачів, які розуміють ризик.

<p align="center"><img src="docs/images/runtime-settings.png" alt="Параметри роботи з режимом керування та режимом роботи колектора" width="480"></p>

Ви можете змінити це пізніше через **Параметри роботи**. Сторінка пристрою колектора призначена для мережевих дій і усунення проблем; пристрій інвертора — це місце для щоденної телеметрії.

## Допомога від хмари SmartESS

Коли локальне виявлення не дає повної впевненості і вдається знайти лише колектор, EyeBond Local може додатково звернутися до хмари SmartESS для того самого ідентифікатора колектора й зберегти повторно використовувані JSON-дані у `/config/eybond_local/cloud_evidence/`.

- **Облікові дані використовуються лише для живого запиту** — інтеграція не зберігає постійний сеанс у хмарі SmartESS.
- **Збережені хмарні дані покращують діагностику та планування метаданих** — саме по собі це не відкриває локальне керування записом.
- **Створити архів підтримки** може автоматично використати вже збережені дані або оновити їх прямо під час побудови ZIP.
- **Експортувати хмарні дані SmartESS** — це окрема розширена дія, якщо вам потрібні самі дані для перегляду чи роботи з експериментальними метаданими.
- **Файли з цими даними лишаються на диску, доки ви не видалите їх вручну**. Для запису автоматично використовується найновіший відповідний файл.

## Як отримати допомогу

Якщо щось не працює, найшвидший шлях такий:

1. Відкрийте екран **Налаштувати → Діагностика та експериментальні метадані** для цієї інтеграції.
2. Натисніть **Створити архів підтримки**.
3. Відкрийте [звернення на GitHub](https://github.com/groove-max/ha-eybond-local/issues) і прикріпіть згенерований ZIP.

Архів підтримки містить анонімізований знімок стану інвертора, читання регістрів і результати розпізнавання. Якщо відповідні хмарні дані SmartESS уже збережені, архів підхопить їх автоматично, а на тому самому екрані ці дані можна оновити прямо перед записом ZIP. Зазвичай цього достатньо, щоб зрозуміти, чи підтримується пристрій, і що робити далі. Якщо ваш пристрій виявиться іншим протоколом або нестандартним варіантом, може знадобитися більше даних або більше однієї ітерації, перш ніж вдасться додати підтримку.

### Шаблони звернень

- **Звіт про помилку** (`Bug Report`) — для відтворюваних регресій на вже підтримуваному обладнанні.
- **Архів підтримки / Діагностика обладнання** (`Support Archive / Hardware Diagnostics`) — для непідтримуваного обладнання, збоїв під час початкового налаштування і часткової підтримки. Завжди прикріплюйте згенерований ZIP.
- **Запит на покращення** (`Feature Request`) — для нової підтримки обладнання або покращення зручності використання.

## Усунення проблем

| Проблема | Що спробувати |
|---|---|
| Автопошук нічого не знаходить | Спочатку скористайтеся пунктом **Змінити інтерфейс сканування** і виберіть інший мережевий інтерфейс. Якщо швидке сканування все одно нічого не дало, запустіть **Глибоке сканування** з екрана результатів. Якщо зрештою переходите до **Ручного налаштування**, спочатку знайдіть локальну IP-адресу Wi-Fi-модуля або колектора, зазвичай у вашому роутері. |
| Налаштування Wi-Fi через Bluetooth недоступне або нестабільне | Якщо на хості Home Assistant немає локального BLE-адаптера або колектор досяжний лише поблизу, додайте **ESPHome Bluetooth Proxy** поруч із колектором і повторіть налаштування Wi-Fi через Bluetooth. |
| Пристрій зависає в **EyeBond Setup Pending** | Це проміжний стан, а не остаточна помилка. Зачекайте трохи, повторіть сканування або ручну перевірку, а якщо колектор так і не підключається назад до Home Assistant або інвертор не вдається визначити, створіть архів підтримки. |
| Зависає на статусі **Лише колектор** | Колектор відповів, але інтеграція ще не може впевнено визначити протокол, профіль або точну модель інвертора. Надішліть архів підтримки. |
| Сенсори лишаються недоступними | Переконайтеся, що колектор у тій самій підмережі, що й Home Assistant, і що TCP `8899` / UDP `58899` не заблоковані. |
| Застосунок SmartESS перестав показувати дані в реальному часі | Перевірте режим роботи колектора. `HA only` навмисно відключає цей колектор від хмари SmartESS. Переключіть назад на `SmartESS + HA`, якщо хочете й далі користуватися штатним застосунком. |
| SmartESS усе ще працює, але Home Assistant каже, що колектор недоступний | Часто це означає, що колектор тимчасово випав із локальної мережі після проблеми з Wi-Fi. Зачекайте кілька хвилин, переконайтеся, що Wi-Fi знову стабільний, і перевірте, чи колектор знову з'явився в локальній мережі. Застосунок SmartESS може відновитися швидше, ніж локальне підключення Home Assistant. |
| Запис наче прийнявся, але значення одразу повернулося назад | EyeBond Local тепер вважає це явною помилкою перевірки після запису, а не мовчазним успіхом. Перевірте в діагностиці лічильники відключень і перезапусків колектора, тимчасово зупиніть паралельні зміни через SmartESS або штатний застосунок і повторіть спробу, коли колектор стабілізується. |
| Віддалений колектор відповідає, але не підключається назад | Насамперед перевірте **Advertised callback IP** і **Advertised callback TCP port**. Вони мають збігатися з реально досяжною для колектора адресою та переадресованим TCP-портом. |
| Віддалене налаштування нестабільне через публічний інтернет | За можливості використовуйте VPN замість прямого NAT, особливо коли одна зі сторін за CGNAT або коли переадресація UDP чи TCP працює ненадійно. |
| Немає елементів керування | У режимі **Auto** керування з'являється лише тоді, коли інтеграція достатньо впевнена у визначенні пристрою і коли ці можливості вже перевірені. Деякі внутрішні режими роботи навмисно лишаються тільки для читання, наприклад резервний режим для родини SMG. Якщо моніторинг уже працює для інвертора родини PI30, але точну модель не зіставлено, можна відкрити **Параметри роботи** і переключитися на **Full Control**. Це ручний обхід захисту, який відкриває всі команди запису, тож використовуйте його лише на власний ризик і бажано після експорту архіву підтримки. |

## Структура репозиторію

Коротка орієнтація для тих, хто переглядає код. Повні нотатки для розробників знаходяться в [CONTRIBUTING.md](CONTRIBUTING.md).

- `custom_components/eybond_local/` — вихідний код інтеграції
- `custom_components/eybond_local/profiles/` — декларативні метадані можливостей у форматі JSON
- `custom_components/eybond_local/register_schemas/` — схеми регістрів для читання у форматі JSON
- `docs/` — публічна документація та згенеровані звіти
- `.github/` — CI, валідація та автоматизація релізів
- `.local/` — нотатки лише для мейнтейнерів, приватні утиліти та локальні release/debug-артефакти, які навмисно не входять до публічної документації
- `.local/fixtures/catalog/` — локальні фікстури для відтворення, що не потрапляють у git
- `tests/` — модульні й регресійні тести
